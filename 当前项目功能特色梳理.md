- 当前这份文档只梳理**现有主项目**（`frontend` + `backend`）已经具备的能力，供后续改新落地页时提炼卖点、功能区块和叙事顺序使用

## 当前产品一句话概括

这不是一个单纯的“AI 出图页”，而是一个带有**账号体系、个人素材库、项目看板、多人可复用的海报生成工作台、生成后精修链路、管理后台**的完整海报生产系统。

## 前端当前可感知的功能特色

### 1. 完整的产品路径，不只是单次生成

- 有独立的落地页、登录/注册页、Dashboard、Board 工作区、Personal Space、Admin 区
- 用户不是“生成一张图就结束”，而是围绕**项目**持续积累与迭代
- 每个项目支持打开、删除、重命名、持续保存、再次编辑

### 2. 项目制海报生产

- 用户先创建项目，再进入画板式工作区进行生成
- 一个项目内可以批量生成多张海报，而不是一次只产一张
- 生成时会先创建占位画板，再后台异步出图，用户能看到生产过程
- 生成结果会以多个 artboard 形式排列，便于横向比较方案

### 3. 画板式创作体验

- 工作区不是表单页，而是接近设计板/白板的交互方式
- 支持画布平移、缩放、选择、多画板布局
- 生成链路会把“需求说明、参考图、Logo、字体参考”等输入，以素材卡片形式挂到画布上，并与结果画板建立连接关系
- 这种表现方式很适合对外表达为“从需求到结果的可视化创作流”

### 4. AI 先策划，再出图

- 不是直接把一句 prompt 扔给模型
- 系统会先把用户需求拆成多个海报方案，包括：
  - 标题
  - 副标题
  - 信息区块
  - 强调色
  - 视觉方向描述
- 之后再按方案批量生成海报
- 这意味着产品卖点可以强调“AI 创意策划 + AI 视觉生成”，而不只是“文生图”

### 5. 可控生成，而不是纯随机生成

- 支持上传参考风格图，并且有强弱档位（low / medium / high）
- 支持选择 Logo，并在生成时要求模型保留和合理放置 Logo
- 支持两类字体控制：
  - 站内可选字体
  - 用户上传的字体风格参考图
- 支持把风格图、Logo、字体参考组合使用，让生成更稳定、更接近品牌或活动要求

### 6. 个人素材库（Personal Space）

- 用户有独立的 Personal Space 页面，用来长期沉淀自己的创作资产
- 目前可管理的资产包括：
  - Reference Styles：参考风格图
  - Font References：字体风格参考图
  - Logo Assets：Logo 资产
  - Primary Colors：主色组
- 这部分非常适合在落地页里被包装为：
  - “你的视觉资产库”
  - “让 AI 越用越懂你”
  - “建立可复用的品牌生成上下文”

### 7. 生成后可继续精修，不是一次性结果

- 支持对已生成海报进行 refinement
- 支持用户输入文字反馈，让系统做增量调整
- 支持 AI 自动给出 4-6 个“下一步创意方向”建议，帮助用户继续迭代
- 支持一次生成多个 refinement variation，便于对比不同版本

### 8. 支持带标注的局部修改

- 用户可以在海报上做编号框/箭头类标注，再配合指令发起精修
- 系统会把“原图 + 带标注图 + 修改说明”一起送给模型
- 输出结果要求去掉标注痕迹，只保留修改后的正式海报
- 这是很强的产品特色，落地页里可以作为“像和设计师沟通一样修改 AI 海报”

### 9. 文案与排版可以二次控制

- 系统支持文本框布局、字号、颜色、字重、对齐等排版信息
- 支持调用后端字体排版接口生成真实文字布局图层
- 支持先生成无字背景，再叠加文字版式，再把文字重新合入最终海报
- 这意味着项目并非只会“整图重抽”，而是正在形成一条更专业的“背景 / 版式 / 合成”链路

### 10. 多尺寸延展能力

- 已支持从现有海报生成更多分辨率/尺寸版本
- 逻辑上强调尽量保持内容、文字、色彩和构图不变，仅做尺寸适配
- 这一点很适合包装成“一个主视觉，多端延展”

### 11. 项目图库与资产沉淀

- 每个项目可以查看已生成图片列表
- 用户可以回看项目内资产，继续点开用于后续迭代
- 项目支持导出 ZIP，包含 manifest 和相关图像资产，便于迁移和归档

### 12. 自动保存与任务恢复

- 项目会定时自动保存到后端
- 同时保留本地兜底存储逻辑，后端不可达时可回退到 localStorage
- 如果页面刷新时仍有任务在跑，系统会恢复轮询并继续拿最终结果
- 这是典型的“更像生产工具、没那么像演示 Demo”的能力

### 13. 新手引导与可学习性

- Dashboard、Board、Personal Space 都做了 onboarding / guide
- 说明产品已经在考虑第一次使用的引导成本
- 未来落地页可以顺势强调“上手快，但能力深”

### 14. 管理后台

- 管理员可进入独立 Admin 区域
- 目前已包含：
  - 数据库表查看
  - schema 查看
  - 行级增删改
  - 用户注册
  - 模型 Playground
- 这说明项目不仅面向终端用户，也已经考虑运维和内部管理

## 后端当前功能特色

### 1. 账号体系完整

- 基于 FastAPI
- 支持注册、登录、刷新 token、退出登录、修改密码
- access token + refresh token 组合
- refresh token 有单独表存储，并支持撤销
- 密码使用 Argon2 哈希
- 支持管理员权限判断
- 支持“首次登录必须改密码”这类企业化流程

### 2. 项目数据与用户隔离

- 项目存储在 SQLite 中
- 项目按用户隔离
- 项目数据包含完整画板、素材、连接关系、视图状态等
- 服务端会把项目中的 base64 图片落盘，转成文件路径，减轻数据库负担

### 3. 用户资产存储能力完整

- Logo、参考风格图、字体参考图都有独立上传与列表接口
- 文件按用户目录隔离
- 上传时会统一做图像校验和转码
- Logo 会同时生成 PNG / WEBP
- 参考图和字体参考图会保存原图路径与缩略图路径

### 4. 字体预览与排版服务

- 后端提供字体列表接口
- 提供字体预览图接口
- 提供文本布局渲染接口 `/fonts/layout`
- 这让前端不只是“选字体名”，而是可以获得实际排版结果

### 5. AI 任务支持同步与异步两套模式

- 有直接聊天/出图接口
- 也有异步任务提交、状态查询、待处理任务查询接口
- 异步任务状态包含 pending / running / completed / error
- 后端用任务表记录任务生命周期，前端可恢复轮询
- 这套能力对“批量生成、多方案生成、刷新后恢复”非常关键

### 6. 图像编辑链路不只做生成

- 后端支持图像编辑接口 `/ai/image/edit`
- 可用于：
  - 去字保留背景
  - 合并版式文字
  - 带标注图修改
  - 尺寸扩展
  - 纯 prompt 出图
- 也就是说，系统本质上有一条通用的“图像编辑工作流引擎”

### 7. 双 provider 兼容思路

- 当前后端既支持传统 Polo 路径，也兼容腾讯云 VOD/COS 图像任务链路
- 说明系统并没有把能力死绑在单一路由上
- 对外叙事上可转化成“可接入企业私有化或多供应商生成基础设施”

### 8. 管理与数据运维能力

- 提供数据库表、结构、分页数据查询接口
- 支持管理员新增、修改、删除数据行
- 这类能力对 B 端产品、内部团队工具、运营支撑都很重要

## 适合提炼到新落地页的核心卖点

### 可以重点强调的 6 个方向

1. **从一句需求到多张成品海报**
   - 不只是出一张图，而是先策划再批量生成多个方案

2. **让 AI 参考你的品牌资产**
   - 风格图、Logo、字体参考、主色组一起参与生成

3. **像在白板上推进设计，而不是在表单里碰运气**
   - 可视化画板、素材输入、结果连接、版本延展

4. **生成之后还能继续改，而且改得更具体**
   - 文本反馈精修、创意建议、标注式修改、多版本 refinement

5. **一张主视觉，继续扩展到更多尺寸**
   - 更适合活动海报、校园宣传、品牌传播、多端投放

6. **它更像一个持续使用的创作系统，而不是一次性工具**
   - 项目制、个人素材库、自动保存、任务恢复、后台管理

## 当前值得注意的“可讲但别讲过头”的点

- `design-guidance` 相关后端表和接口已经存在，但当前前端主流程里还没有完全打成明显功能位，落地页里可以谨慎表述为“可扩展的设计指导能力”，不要直接说成成熟主功能
- 新参考落地页目录目前只是存在于 `for reference`，根据当前主项目代码梳理，它还没有接入现有正式前端主流程
- 现有主项目真正最强的差异化，不是视觉包装，而是**可控生成 + 生成后迭代 + 项目化沉淀**

## 给后续改新落地页的建议

- 页面叙事不要只写“AI 海报生成”
- 更建议围绕下面这个结构来组织：

1. Hero 区先讲结果
   - 例如：几分钟产出多版高质量海报

2. 第二屏讲可控性
   - 风格图、Logo、字体、主色组如何让结果更稳定

3. 第三屏讲工作流
   - 从需求输入，到方案生成，到画板管理，到 refinement

4. 第四屏讲持续使用价值
   - Personal Space、项目保存、多尺寸延展、团队管理

5. 第五屏讲适用场景
   - 校园活动、社团海报、演出宣传、品牌活动、系列 KV 延展
